开营前的准备 / Before we start

  • 注册一个 GitHub 账号，并把用户名发给助教
  • 安装 Cursor（或你习惯的 AI 编程工具），并登录
  • 安装 飞书 Lark——第二天田野调查录音要用，可转写 3 小时
  • 一台能上网的笔记本电脑（不需要安装 Git，也不需要用命令行）
  • 好奇心，愿意动手，以及不怕东西一开始跑不起来的耐心

你会学到什么 / What will I learn

你会完整走一遍产品创造者的路：从真实用户身上发现问题，写下一页产品规格， 指挥 AI 把它做出来，测试它、攻击它，最后把它发布出去，并讲清楚自己为什么这样做。

技术上，你会掌握 Vibe Coding——用清晰的语言驱动 AI 编程；学会用 GitHub 保存和分享代码； 用飞书多维表格作为真实的后端；给 AI 写 SKILL.md 让它稳定地做好一件事。

但这门课真正想留给你的，是另外几样东西：学会质疑并检验 AI 给的答案（它会一本正经地说错）、 在不确定中敢于开始、在反复失败时保持耐心，以及把别人的批评当作自己前进的燃料。

贯穿全营的一条主线：Model → Agent → Workflow。 第一天你自己是那个循环——你运行代码、你把报错贴回去；第四天，AI 智能体自己完成这个循环； 第六天，你会看到多个智能体自己决定谁该做什么。

每日课程 / Daily notes

DAY 1 注意力战争，与动手搭建 Model

今天的目标

理解那些让人停不下来的 App 究竟是怎么设计的；搞清楚 AI 为什么会一本正经地说错话； 并亲手做出你的第一个能运行、能发布的应用。

流程

  • 破冰："你站在哪一边"——用身体投票，为自己的立场辩护
  • 轮流分享：你平时怎么用 AI？它有没有骗过你？
  • 迷你讲座：AI 幻觉是怎么产生的——它预测下一个最可能的词，而不是查资料
  • 讲座：注意力经济、成瘾式设计，以及各国的监管回应
  • 开放思考：画一张思维导图，找到你想解决的问题
  • 技术讲座：什么是 Vibe Coding（Model → Agent → Workflow 全景图）
  • 动手搭建：用 AI 做出一个可点击的 demo
  • 发布上线：推送到 GitHub，拿到你的第一个网址
  • 组队，并确定每组要去田野验证的问题

今日金句

"会编造假事实的 AI，也一样会编造不存在的函数。"

DAY 2 田野调查：江西上饶 Field

今天的目标

见到真实的用户。看着他们使用你昨天做的东西——这是你能得到的最诚实的反馈。

流程

  • 火车上：确定访谈问题、分工、测试 Lark 录音。两人一组，一人问、一人记
  • 抵达：参观校园、破冰游戏、午餐（众谷主持）
  • Part 1：由我们的学生为乡村同学主持"你站在哪一边"
  • Part 2：Demo 实测——乡村同学真实使用你的产品，你在旁边观察记录
  • Part 3：知识挑战小竞赛
  • Part 4：心愿墙——"我希望 AI 能为我们校园做 ___"，每一条都拍下来
  • 家访：每组一个家庭，只做访谈
  • 返程火车上：把发现变成产品结论

心态

我们是倾听者，不是拯救者。今天真正的工作是听。

DAY 3 产品设计与伦理审计 Spec

今天的目标

带走三样东西：一份宪法、一页产品规格、一个能点的 demo。

流程

  • 规格讲座：现场对比——同一个 AI，模糊的提示 vs. 清晰的规格，结果天差地别
  • 写你的规格：用七个问题，或让 AI 反过来采访你，然后你来删改
  • 伦理讲座（合上电脑）：三种视角——后果、权利与义务、德性
  • 伦理审计 → 写下你的宪法：把审计结论变成 5–7 条永不逾越的规则
  • 从规格到 demo：用 mock 数据做出一个核心功能
  • 董事会路演：其他组当董事会，任务是——找出你的产品会违反自己哪一条规则

今天最重要的一句话

"没有机制保障的规则，就是你迟早会违反的规则。"
在自己的产品里发现漏洞，是加分，不是丢脸。

DAY 4 从 Demo 到 MVP：让表格成为后端 Agent

今天的目标

今天你从"自己写代码"升级为"指挥智能体写代码"。今天必须跑通的一件事： 应用能从后端读出故事，并把一次投票写回去。

流程

  • 技术讲座：仓库，以及那两个指挥智能体的文件——clone / commit / push； 开源与许可证；CLAUDE.md 与 SKILL.md（不是代码，是指令）
  • 计划 + 数据结构：在投屏的飞书多维表格上现场建表—— 一行一个记录、一列一个含义、每行都要有 ID（主键）、投票放新的一行（一对多）
  • 连接 + 冒烟测试：先证明数据管道是通的，再写任何产品代码
  • READ：故事出现在你的应用里
  • 开源清单：看看别人已经做过什么，克隆一份示例 SKILL.md
  • 写一个真正好用的 Skill，然后做出生词表功能——本产品唯一的 AI 核心功能
  • WRITE：投一票，看着新的一行实时出现在投屏的表格里
  • 《开发者日志》：今天智能体哪里错了？你改了什么？

你的宪法，现在开始生效

把宪法放进 CLAUDE.md 的最上面——智能体在写第一行代码之前就会读到它。

DAY 5 先让它能用，再让它诚实 Agent+

今天的目标

上午：把产品真正做完、做稳。下午：让它对用户诚实，然后亲手攻击它。

流程

  • 技术讲座：测试与调试——重现 → 定位 → 修复。报错是正常的，也是可读的
  • 把功能做完，对照验收清单逐条检查
  • 橡皮鸭复查：先说清楚这段代码"应该做什么"，再让 AI 读代码，找出实际做了什么
  • 交付关卡：每组的网址必须能在众谷教室的电脑上打开（午餐前硬性通过）
  • 技术讲座：信任与透明——问 AI"你为什么这么写"，它会编一个听起来很合理的理由
  • 做透明面板：展示真实存下来的东西（来源、解释了哪些词），而不是事后编的说法
  • 把宪法变成代码：生词必须真的出现在原文里、数量必须在范围内、未经审核的不许显示……
  • 红队：攻破你自己的宪法。与另一组互换产品，失败都记为"违反第 N 条"

本周最好玩的实验

AI 会给自己的每条输出打一个"信心值"。去验证一下：它的信心真的能预测它有没有错吗？ 多半不能——因为那只是它讲给你听的故事，不是证据。

DAY 6 实测、冻结、讲好你的故事 Workflow

今天的目标

在真实的乡村网络条件下测试产品，锁定版本，然后开始打磨路演故事与伦理案例。

流程

  • 技术讲座（前瞻）：Agentic ≠ 多智能体。 现场演示 Claude Code 多智能体协作——没有人告诉它顺序，它自己读描述决定谁来做
  • 盲测：互换产品，把网络限速到 Slow 3G、CPU 降速、切到手机屏幕，然后完全无人指导地使用
  • 修复真正会挡住用户的问题，然后 CODE FREEZE 代码冻结——之后不再加新功能
  • 讲座：AI 的发展历程
  • 叙事工作坊 + 路演训练：问题、用户、产品、证据、诉求
  • 伦理案例写作工作坊（Marc Sanders 案例大赛）

诚实的结尾

"刚才那件事用了四次模型调用，而一个智能体就能做完。更多的智能体意味着更贵、更慢、更多出错的地方。 真正的工程师，能用一个就不用两个。"

DAY 7 Demo Day 路演 Ship

今天的目标

把作品公开发布出去。你带走的是：一个在线可访问的应用、一份《开发者日志》、一份伦理案例—— 这就是你的作品集。

流程

  • 嘉宾分享：伦理与教育
  • 技术彩排：在已冻结的版本上完整走一遍，并准备好录屏作为备份
  • 评委实测：评委亲自使用你的产品并给出反馈
  • Demo Day 路演：10 分钟展示 + 5 分钟问答
  • 颁奖：最佳技术架构奖（$1,000 种子基金）、最具人文关怀奖（Marc Sanders 案例大赛通道）

规则

只修演示，不修产品。代码已经冻结了。
